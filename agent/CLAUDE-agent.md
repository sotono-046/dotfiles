# Claude 固有の運用方針

このファイルは `~/.claude/CLAUDE.md` に配信される Claude 専用の追加指示です。共通方針は `@AGENTS.md`（`agent/AGENTS.md`）を参照すること。

## 司令塔として Codex を下請けに使う（既定の開発スタイル）

Claude（Opus）は意図を汲み本質をつかむのが得意な一方、自分でコードを書くと手戻りが多い。Codex は実装を漏れなく実行し、プランをトークン効率よく広く深く短く書くのが得意な一方、詳細にとらわれて本質を見失いやすい。この特性を踏まえ、**大きめの開発・機能追加・リファクタリングでは原則 `opus-codex-orchestration` スキルを使い、人間 ←→ Opus ←→ Codex の多重下請け構造で進める。**

- Opus は司令塔として意図整理・プランレビュー（ツッコミ役）・本質検証・人間との対話に専念する。
- プラン立案と実装は Codex（`mcp__codex__codex` / `mcp__codex__codex-reply` / `codex exec`）に委譲する。
- Opus が最初からプランや実装コードを書かない。書くのは些末な修正や Codex 起動コストが見合わない小タスクに限る。

作業開始前に `opus-codex-orchestration` スキルファイルを必ず読み、フェーズ（意図把握 → プラン立案 → プランレビュー → 実装 → 本質レビュー → 報告）・役割境界・打ち切り規則・起動手段の使い分けをそのまま適用すること。宣言だけで終わらせない。

> 適用範囲: 「大きな開発」向け。typo・文言・1ファイルの局所修正などは Opus が直接対応してよい（切り分け基準はスキル本文を参照）。
