# 8 ポイントグリッドによるスペーシング設計

UI の「なんとなく崩れて見える」は、ほぼ縦／横方向の **余白リズム** で決まる。
中核ルールは 1 つ ── **すべての高さ・余白・gap を 4 の倍数にし、基本は 8 の倍数を使う**。
これでデバイス密度に強くくっきりし、デザインとコードが同じ単位を話す。本スキルは
それを「縦に積むリスト系 UI」を中心に、フォーム・カード・モーダル・ツールバーまで
一貫した方法で適用するための指針。

## 適用範囲

このスキルは「要素間の余白で意味（グループ／階層）を伝える UI」全般で使う。代表例:

- **サイドバー／ナビゲーションレール／アプリシェルの左メニュー**（最も典型。下のテンプレ参照）
- **設定画面・プロフィール画面のリスト**（セクション分けされた縦リスト）
- **フォーム**（ラベル↔入力、入力間、フィールドグループ、送信ボタン周辺）
- **カード／パネル**（内部 padding、ヘッダ↔本文↔フッタ、カード間 gap）
- **モーダル・ダイアログ**（コンテンツ余白、アクション行の上 gap）
- **ツールバー・ヘッダー・タブバー**（横方向の項目 gap / グループ間 gap）
- **テーブル行・タイムライン・通知リスト**（行高、行間、グループ間の隙）
- **チップ／タグ群、ボタングループ**（横方向の小 gap）

縦・横どちらでも同じ原則（**項目間 gap < グループ間 gap**）が効く。以下「行高」「項目 gap」
「セクション gap」と書いた箇所は、横並び UI では「セル幅」「項目間 gap」「グループ間 gap」と
読み替える。

## まずアンカー: サイドバー仕様テンプレート（代表例）

新規でも改修でも、リスト系 UI はこの縦リズムを出発点にする。値はすべて 4/8 グリッド上。
**サイドバー以外（設定画面・通知リスト・フォーム）でも同じ数値ファミリーで組める** ので、
まずこれを暗記する。

```
Container（展開 ≈260–280px 幅）
├─ 44  上 padding                          端の余白 / タッチゾーン
├─ 32  ロゴブロック
│   16 gap
├─ 32  検索フィールド領域
│   16 最初のセクションまでの gap
├─ Section "Main"
│   16  セクションラベル → 最初の項目
│   40  ナビ項目の高さ（24px アイコンを中央 → 40 のクリック領域）
│    4  項目間 gap            ← グループ内（詰める）
│   40  項目…（Orders / Documentation / Map / Statistics）
│   24  次セクションまでの gap  ← グループ間（広げる）  ★ 4 < 24
├─ Section "Communication"（Inbox / Couriers）── 同じ 40/4 リズム
├─ 48  カードまでの gap
├─ カード "Used capacity"（高さ ≈208 = 8×26、内部 padding は 8/12/16/32）
│   56  進捗 / ヘッダーブロック
├─ 48  フッターまでの gap
├─ Section フッター（Settings / Help）── 40/4 リズム
├─ プロフィール行（周囲 16/16）
└─ 44  下 padding
```

このテンプレが体現している 3 点を必ず守る。

- 行高 **40** ・ 項目 gap **4** ・ セクション gap **24** が、Main / Communication / フッターを別ブロックに見せている
- **44** の上下だけが唯一の非 8 倍数。44 のタップターゲット標準に合わせた意図的な例外で、これ 1 つは可。例外だらけは不可
- 折りたたみ版は **同じ縦リズム** を保ち、ラベルを落としてホバー時にツールチップで見せるだけ

## 値のスケール（トークン）

UI で使う値はこの範囲にほぼ収まる。生数値ではなくトークン名で出す。サイドバー以外の
用途（フォーム・カード・モーダル）でも同じトークンを共有する。

| トークン      | px  | =    | 代表的な用途 |
|--------------|-----|------|--------------|
| `xs`         | 4   | 4×1  | **同一グループ内**の項目間 gap（リスト行間、チップ間） |
| `sm`         | 8   | 8×1  | カード／フォーム内部の詰めた gap、アイコン↔テキスト |
| (half)       | 12  | 4×3  | カード内テキストの微調整、密集テーブル行間のみ |
| `md`         | 16  | 8×2  | ラベル↔入力、検索→セクション、カード内 padding、プロフィール周囲 |
| `lg`         | 24  | 8×3  | **グループ／セクション間**の区切り、フィールドグループ間 |
| `xl`         | 32  | 8×4  | ロゴ／検索ブロック領域、モーダル／カードの padding（ゆったり）、フォームのタイトル↔最初のグループの上限 |
| `row`        | 40  | 8×5  | リスト行 1 行の高さ（クリック領域）、フォーム入力高さ、ボタン高さの基準 |
| `edge/touch` | 44  | 4×11 | コンテナ上下 padding / タップ標準（iOS HIG）／タッチ・マウス両対応時の行高 |
| `gap-lg`     | 48  | 8×6  | カードやフッターブロックへの大きな gap、モーダル↔アクション |
| `block`      | 56  | 8×7  | カード上部の背の高い要素、フォームのプライマリブロック |

### 入力デバイス別の `row` / `edge` 選択

| 主入力デバイス | `row` | `edge` | 例 |
|----------------|-------|--------|----|
| マウス主（デスクトップ） | **40** | 44 | 業務ダッシュボード、PC アプリ |
| 両対応（ラップトップ + タッチ） | **44** | 44 | 一般的な Web サイト |
| タッチ主（モバイル／タブレット） | **48** | 48 | スマホアプリ、Material 寄り |

迷ったら **44**。原則（4/8 倍数）は崩れない。

### 12px (half) の使用条件 — 限定 2 ケースのみ

12 は 4 の倍数だが **デフォルト不可**。次の 2 ケースに限り許容する。それ以外は 8 か 16 にスナップする。

- ✅ カード／パネル内のテキスト微調整（行間調整・キャプション↔本文 など）
- ✅ 密集テーブルの行間（標準 16 では縦が伸びすぎる業務テーブル）
- ❌ 行間 gap、グループ間 gap、padding、カード間 gap、フィールド間 gap などの主要な構造 gap には **使わない**

「8 と 16 のどちらにスナップしても破綻するとき」の最後の砦として残す位置付け。

## ルール

ルールは「サイドバー」を例に書くが、**他の UI でも全部効く**。各ルールの末尾に
他用途への翻訳をつけた。

### 1. 行高はアイコンではなくクリック領域で決める
24px アイコンの見た目サイズはクリック可能サイズではない。アイコン周囲の padding で行全体をターゲットにする。

- **デスクトップ・マウス** … 40px 行高が定番（ポインタは精密）
- **Apple HIG 最小** … 44×44pt ／ **Material 最小** … 48×48dp
- タッチ併用やアクセシビリティ優先なら 44〜48 に寄せる

行をアイコンに縮めない。逆にアイコンが 24px でも行は 40px に保つ。

> 他用途: ボタンの高さ、フォーム入力欄の高さ、テーブル行、リストアイテム、タブにも同じ閾値を適用する。

### 2. 項目間 < セクション間 にする（最重要）
余白は装飾ではなく情報。近接により、近い項目は 1 グループ、離れた項目は別物と読まれる。次の不等式を必ず守る。

> 項目間の gap  <  セクション間の gap

テンプレなら **4 < 24**。これが崩れると「全部の項目がのっぺり繋がって、どこからどこまでが一塊か分からない」状態になる。レイアウトが崩れて見えるときの最大の原因はここ。

> 他用途（隣接 2 段の不等式が常に成り立つこと）:
> - フォーム → label↔input (8) < field 間 (16) < field group 間 (24) < title↔body (24〜32)
> - カード一覧 → カード内ブロック間 (16) < カード間 (24)
> - カード内（3 段入れ子）→ アイコン↔テキスト (8) < ヘッダ↔本文 (16) < 本文↔フッタ (24)
> - モーダル → 本文段落間 (8〜16) < 本文↔アクション行 (24〜32)
> - ツールバー → 関連アイコン間 (4〜8) < アイコングループ間 (16〜24)

#### 階層が 3 段以上のとき

カードのように「アイコン / テキスト / ブロック / カード / カード一覧」と階層が深いケースでも、ルールは単純: **隣接する 2 段の間で必ず `内側 gap < 外側 gap` を満たす**。トークンスケール（4 → 8 → 16 → 24 → ...）を、深さに応じて 1 段ずつ上げて使えば自動的に成立する。同じ値を 2 段連続で使うと、その境界が消える。

#### 不等式の対象は「要素間 gap」のみ — padding/inset は対象外

不等式 `内側 < 外側` の対象は **要素と要素の間の gap / margin**。コンテナの **padding（inset／端の抜き）はルール 4 の管轄** で、不等式の階層計算には含めない。よって `card padding = 24` と `card 間 gap = 24` が同値でも不等式違反ではない（前者は外周の抜き、後者は要素間の gap）。同様に **1px hairline 罫線・1〜2px の境界線は装飾扱いで gap に含めない**。

### 3. 見出し／セクションラベルは下のコンテンツに寄せる
"Main" "Communication" などのラベルは、上のセクションから離し（24）、自分が束ねる項目には近づける（16）。ラベルが上下等間隔だと、どちらのグループの見出しか曖昧になる。

> 他用途: フォームの section heading、設定画面のグループ見出し、カードの内部見出し、モーダルのサブヘッダにも同じ「上に広く・下に狭く」を適用する。

### 4. コンテナの端は十分な padding で抜く
サイドバーなら上下に 44。カードなら内側に 16〜24。モーダルなら 24〜32。**端の要素が縁に貼り付かない** ことが目的。44 はタップ標準でもあり、唯一許される非 8 倍数の例外。

> 他用途: カード内 padding は 16（密）／24（標準）／32（ゆったり）の 3 段から選ぶ。モーダルは 24 か 32。ページコンテンツは 24〜48。

#### 垂直 edge と水平 inset を分けて考える

- **垂直方向（上下 padding）**: 用途で値が変わる。
  - サイドバー／アプリシェル → **`edge = 44`**（タップ標準を兼ねる）
  - 大型モーダル（幅 ≧ 480px、フルダイアログ）→ **24 か 32**
  - 小型モーダル／popover／ドロップダウン（通知センター、メニュー、ツールチップ大）→ **16 か 24**（44 は使わない。密度優先）
  - カード → **16（密）／24（標準）／32（ゆったり）**
- **水平方向（左右 padding / inset）**: 独立した語彙で扱う。`inset-h: md(16) / lg(24) / xl(32)` の 3 段から、コンテナ幅に応じて選ぶ。
  - 幅 ≦ 320px（狭いサイドバー / モバイル幅）→ **16**
  - 幅 320〜640px（標準カード / 設定画面）→ **16 or 24**
  - 幅 ≧ 640px（ページコンテンツ / ワイドモーダル）→ **24 or 32**

垂直 edge をそのまま水平にも使わない。44 を左右に使うと縁が間延びする。

### 5. 状態変化（折りたたみ／レスポンシブ）で縦リズムを変えない
アイコンのみ表示にしても、行高 40・項目 gap 4・セクション gap 24 はそのまま維持する。変えるのは幅とラベルの有無だけ。ラベルはホバーのツールチップで補う。展開↔折りたたみで項目の y 座標がズレないのが理想。

> 他用途: モバイル時の縦積みカード、テーブル → カード変換、ダークモード切替などでも、**間隔のトークンは同じものを使い回す**。breakpoint ごとにマジックナンバーを足さない。

### 6. トークン化して CSS に直結
要素 *間* の刻みを測るソフトグリッドなので、そのまま `padding` / `margin` / `gap` に落ちる。生数値を散らさず名前付きで出す。

```css
:root {
  /* 汎用トークン（サイドバー以外でも使う） */
  --space-xs: 4px;
  --space-sm: 8px;
  --space-md: 16px;
  --space-lg: 24px;
  --space-xl: 32px;
  --space-2xl: 48px;
  --row-h: 40px;          /* リスト行 / 入力欄 / ボタンの基準高さ */
  --edge: 44px;           /* コンテナ上下 padding / タップ標準   */
  --inset-h: 16px;        /* コンテナ水平 inset（幅で 16/24/32）  */

  /* サイドバー固有エイリアス（用途を読みやすく） */
  --sb-row: var(--row-h);
  --sb-item-gap: var(--space-xs);
  --sb-section-gap: var(--space-lg);
  --sb-edge: var(--edge);
  --sb-inset: var(--space-md);
}
```

Tailwind なら `h-10`(40) / `gap-1`(4) / `gap-6`(24) / `p-4`(16) / `py-11`(44) がそのまま対応。任意値 `[13px]` は使わない。

## フォーム適用テンプレ（よく使うので別出し）

フォームは「label+input というミニ縦スタックを field 単位で並べる」入れ子構造なので、サイドバーより 1 段階層が深い。次の値で組めば不等式が自動で成立する。

```
Card / Container
├─ 32      padding（xl, ゆったり）           ← 端の inset
├─ Title
│   24      title → 最初の field group       ← lg（タイトルは離す側）
├─ Field group "認証情報"
│   ├─ Field "Email"
│   │   ├─ label
│   │   │    8   label → input               ← sm（field 内、最も密）
│   │   └─ input  高さ 40                    ← row-h
│   │  16  field 間 gap                      ← md
│   ├─ Field "Password" ── 同じ 8 + 40
├─ 24      field group → 補助行              ← lg
├─ 補助行（Remember me / Forgot password）
│       高さ 40 / space-between              ← row-h
├─ 24      補助行 → アクション               ← lg（本文↔アクション）
└─ Submit ボタン  高さ 40                    ← row-h
```

不等式: `8 (label↔input) < 16 (field 間) < 24 (group 間 / title 下 / aux 上下)` を一直線で満たす。

### 補助行（横並びユーティリティ行）

「Remember me + Forgot password」「キャンセル + 保存」のような横並び行は、

- 行全体に `min-height: var(--row-h)` を当てる（クリック領域確保）
- `display: flex; align-items: center; justify-content: space-between`
- 行内項目間 gap は `sm(8)` または `md(16)`、行と前後ブロックの gap は `lg(24)`

縦リズム上の 1 行扱いにして、行高 40 を維持するのがコツ。

## 生成するとき（リスト系・フォーム系・カード系 共通フロー）

1. **トークンを先に決める**（行高・項目 gap・グループ gap・端 padding・水平インセット）
2. **コンテンツを縦順に積む**（サイドバーならロゴ→検索→セクション→カード→フッター→プロフィール／フォームならタイトル→セクション×n→アクション／カードならヘッダ→本文→フッタ）
3. **項目 gap を小さく・グループ gap を明確に大きくする**（ルール 2）
4. **各行は padding でクリック領域を作る**（ルール 1）。フォームなら入力欄も同様。
5. **状態違い（折りたたみ／レスポンシブ／ダーク）でトークンを使い回す**（ルール 5）

## レビュー / 監査するとき

違反値 → グリッド上の値の形で具体的に返す。次の 4 列テーブルを既定フォーマットにする。

| 箇所 | 現在値 | 違反ルール | あるべき値（トークン） |
|------|--------|-----------|----------------------|
| カード内 padding | 10px | off-grid（4 倍数外） | 16 (md) |
| ボタングループ間 | 10px | ルール 2（項目間 = グループ間） | 24 (lg) |
| アクションボタン高 | 28px | ルール 1（クリック領域 < 40） | 40 (row) / 44 (両対応) |

その上で次の観点を全て確認する。

- [ ] すべての行高・gap・padding が 4 の倍数（理想は 8）か。外れた値を列挙
- [ ] 項目間 gap がグループ／セクション間 gap より厳密に小さいか（ルール 2）
- [ ] クリック／タップ可能要素が 40 以上で、領域を要素縮小でなく padding で作っているか
- [ ] 見出しラベルが上に離れ・下のコンテンツに寄っているか（ルール 3）
- [ ] コンテナ端に 16〜44（用途相応の 8 系）padding があるか
- [ ] 状態違いで縦リズムが保たれているか
- [ ] 値がトークン化されているか、マジックナンバーが散っていないか
- [ ] サイドバーの場合: 折りたたみ版が展開版と同じ y リズムを維持しているか

## よくある失敗

- **項目間とグループ間が同じ余白** → グループが見えずのっぺりする（最頻出）
- **ヒット領域を要素に合わせる** → 行／ボタンが細すぎて誤クリック・誤タップを誘発
- **見出しラベルが上下等間隔** → どのグループの見出しか曖昧になる
- **折りたたみ・レスポンシブで余白系を作り直す** → 状態切替で要素位置がズレて落ち着かない
- **`13px` `25px` などの off-grid 値** → 1.5x でぼやける。12/16・24 にスナップ
- **トークンを定義したのに使わない** → 結局生数値が散る。Tailwind の任意値 `[13px]` 禁止
