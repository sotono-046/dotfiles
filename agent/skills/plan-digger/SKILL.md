# plan-digger

コードベースの品質・セキュリティ・パフォーマンスを包括的に検証し、改善計画を策定する。

## 前提

- `codex` CLI が利用可能（`codex exec` で非対話実行する。`codex` コマンドが見つからない場合は `claude -p` のみで実行し、その旨をプラン文書冒頭に明記）

## レビュー観点

下記について厳しくレビューを行う。
また、すべての指摘はコードベースを根拠とすること。

- **品質**: 可読性・重複・責務分離
- **セキュリティ**: 脆弱性の検出（認証・認可、インジェクション、データ漏洩など）
- **バグ検出**: ユーザーが認識していない潜在的なバグ
- **パフォーマンス**: ベストプラクティスに基づく最適化ポイント

## レビュー方法

1. 一次レビュー: `claude -p '<レビュープロンプト>'` で反復実行（重大度 High 以上が 0 件になるまで）
2. 二次レビュー: `codex exec '<レビュープロンプト>'` で非対話的に最終検証（Bash ツール経由で 1 回起動するだけの一時コマンド扱い、MCP サーバー常駐は不要）
3. 二次で新規指摘が出た場合は 1 に戻る
4. 二次承認（指摘 0 件）でプラン文書を確定保存

### codex exec の使い方

```bash
# プロンプトを引数で渡し、結果を標準出力で受け取る
codex exec 'このディレクトリのコードをセキュリティ観点でレビューして High/Medium/Low で分類'

# プロンプトを stdin から渡す（長文や heredoc 向け）
codex exec - <<'EOF'
以下の観点で <対象ファイル> をレビューせよ:
- セキュリティ
- バグ検出
- パフォーマンス
EOF
```

`-c` オプションで `~/.codex/config.toml` の設定を一時オーバーライド可能。永続的な MCP セットアップは不要で、一回の `codex exec` 呼び出しで完結する。

## 承認基準

- High: 0 件必須
- Medium: SOW に記載で許容
- Low: 任意（注記のみ）

## 調査

- 必要に応じて追加調査を行う
- 調査結果は必ずコードベースの該当箇所と紐づけて記録
- 懸念事項など、ユーザーに判断を求める場合は `AskUserQuestion` ツールを使って質問してください。

## 成果物

### プラン文書

- 形式: SOW（Statement of Work）
- `CLAUDE.md` にて指定された保存先がある場合はそれに準じ、ない場合は `.temp/YYYY-MM-DD/<slug>.md`（slug はケバブケース）に保存する。
- レビューごとにブラッシュアップし、最終版を残す

## 終了条件

二次レビュー（`codex exec`）で新たな問題が検出されなくなった時点で終了。

## references 指針

- 現状単一ファイルで完結。手順詳細化が必要になった場合のみ `references/sow-template.md` 等に分離する。
