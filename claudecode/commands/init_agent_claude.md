# AGENTS と CLAUDE の初期化

- `CLAUDE.md`と`AGENTS.md`に書いてある内容を比較し、統合する
- 統合した内容を何度かレビューする
- レビュー結果を`AGENTS.md`に書き込む
- `CLAUDE.md`の中身を下記に書き換える
  ```md
  @AGENTS.md
  ```

## レビュー要件

### 1. レビュー観点

- **セキュリティ**: 脆弱性の検出（認証・認可、インジェクション、データ漏洩など）
- **バグ検出**: ユーザーが認識していない潜在的なバグ
- **パフォーマンス**: ベストプラクティスに基づく最適化ポイント

### 2. レビュー方法

- `claude -p` を使用して反復レビューを実施
- 回数: 3〜10 回（問題が検出されなくなった時点で終了）
- すべての指摘はコードベースを根拠とすること

### 3. 調査

- 必要に応じて追加調査を行う
- 調査結果は必ずコードベースの該当箇所と紐づけて記録

## 成果物

### プラン文書

- 形式: SOW（Statement of Work）
- 保存先: `../.temp/YYYY-MM-DD_(任意の名前).md`
- レビューごとにブラッシュアップし、最終版を残す

## 終了条件

- 反復レビューで新たな問題が検出されなくなった時点
