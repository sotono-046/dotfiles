# セキュリティ調査・対策プラン指示書

## 目的

プロジェクトのセキュリティリスクを特定し、対策プランを策定する。

## 手順

### 1. 最新セキュリティ情報の収集

- Web 検索を使用し、プロジェクトに関連する最新のセキュリティ情報を収集する
  - 使用技術スタック（フレームワーク、ライブラリ等）の脆弱性情報
  - CVE、セキュリティアドバイザリ
  - 業界で報告されている攻撃手法・インシデント

### 2. プロジェクト内リスクの探査

- **コード**: 認証・認可、インジェクション、データ漏洩等の脆弱性
- **パッケージ**: 既知の脆弱性を持つ依存パッケージ
- **バージョン**: サポート終了・非推奨のランタイムやライブラリ
- **依存関係**: サプライチェーン攻撃のリスク、推移的依存の問題

### 3. 情報整理・対策プラン策定

- 収集した情報とプロジェクト内リスクを突合
- 優先度（緊急度 × 影響度）を付けて対策プランを策定
- 形式: SOW（Statement of Work）

### 4. プランレビュー

- `claude -p` を使用して反復レビューを実施
- 回数: 3〜10 回（問題が検出されなくなった時点で終了）
- すべての指摘はコードベースを根拠とすること

## 成果物

- 保存先: `../.temp/YYYY-MM-DD_security-report_(任意の名前).md`
- 内容:
  - 収集したセキュリティ情報のサマリ
  - 検出されたリスク一覧
  - 対策プラン（SOW 形式、優先度付き）

## 終了条件

- 反復レビューで新たな問題が検出されなくなった時点
